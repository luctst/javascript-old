How javascript works

What is an execution stack

The javascript code needs to be executed in a specific environment for it to work and this environment is called the execution context, we can imagine this as a box that contains our code, variables etc. Here where our code is executed.

How code is executed

  • In javascript functions create a scope which means that all codes declared outside a function is associated to the global object and in javascript a browser is global object represented by window object, so if you create a variable in the global object you automaticaly add a property to the window object because variable are like properties in object.
  • So in our example above, we create a variable in our global context so she's stores in the global context, then we declare three functions in the global context so they are stores into this one.
  • After that still in our global context we call the first() function up until this point our execution stack now contain on top the first() execution context which means that now javascript will run the code inside thefirst() function.
  • Now that we are in the first() function execution we store variable a and then we call the second() function so now on top of our execution stack we have the second() function.
  • We call the variable b and then the third() function so execution stack on top is now third() function execution context.
  • And then javascript stores variables c and z, our third function is now done so the third() execution context on the stack is now push outside, we're now back on the second() execution context we store the variable z and then we push out the second() execution context.
  • Back in our first() function context we can now stores the last variable x and push out the first() function context.
